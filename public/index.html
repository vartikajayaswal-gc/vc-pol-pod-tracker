Root Cause Analysis
🔍 Single Root Cause Dominates: 96.5% of alerts stem from GoTrack using terminal-level granular names while GoShipment stores city/port-level names. These are the same physical locations — the mismatch is purely a naming convention gap, not a data error in routing.

Recommendations & Suggested Dimensions
🔧 Immediate Actions
01

Build Port Alias Normalization Table

Create a master mapping of 16 known terminal↔city pairs in GoTrack. Apply normalization before the mismatch check runs. This single fix eliminates 96.5% of all alerts (548/568 CRNs) with zero impact to actual data.

⚡ Impact: -96.5% alert volume
02

Fix 20 CRNs with Missing POL in GoTrack

20 CRNs have no POL set in GoTrack. These are genuine data gaps. Backfill the correct POL and investigate why the tracking link didn't capture the origin terminal.

⚡ Impact: Resolves 8 P0 genuine errors
03

Investigate Cavan ↔ Mississauga Bidirectional Flip

4 CRNs have GoTrack: Davao→Mississauga vs GoShipment: Davao→Cavan, and 4 others have the reverse. This bidirectional inconsistency may indicate a data entry error or port code confusion in one of the systems.

⚡ Impact: Clarifies 20 ambiguous CRNs
04

Add Deduplication Window on goshipment_updated

Alert burst on Jun 24–25 came from the same CRNs triggering multiple times (avg 7.1×/CRN) across a 15.3h window. Implement a 5–10 minute dedup window per CRN per alert type to cut noise from 4,032 firings down to ~568.

⚡ Impact: -8.6% alert noise
